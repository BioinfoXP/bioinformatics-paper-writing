# Learning from Literature

## Purpose

Use this file when the user wants the skill to learn from a paper and keep reusable writing knowledge for future bioinformatics manuscript work.

## Inputs

Accept any of the following:

1. PDF
2. DOI or journal page
3. abstract
4. full text
5. figure legends
6. supplementary methods
7. pasted notes

## Goal

Extract reusable writing guidance, not paper-specific scientific claims.

Always separate:

1. transferable writing pattern
2. domain-specific scientific content
3. uncertain preference or one-off stylistic choice

## Extraction Schema

For each learned paper, capture:

1. citation or identifier
2. disease or biological system
3. study design and cohort structure
4. modality stack
5. Results ladder
6. paragraph-opening patterns
7. evidence verbs and caution verbs
8. validation language
9. Methods reproducibility phrasing
10. Discussion framing
11. figure legend compression pattern
12. reviewer-risk signals

## Saving Rules

Save one structured note per paper under `knowledge/paper-notes/`.

Suggested file name:

`YYYY-MM-DD-first-author-short-topic.md`

If a stable pattern appears in multiple learned papers, summarize it in `knowledge/syntheses/`.

## What Not to Generalize

Do not promote these directly into reusable guidance without qualification:

1. disease-specific mechanistic claims
2. one paper's preferred terminology when it is field-specific
3. statistical thresholds that are study-specific
4. claims that depend on unavailable supplementary evidence

## Conflict Handling

When a new paper disagrees with saved knowledge:

1. record the conflict
2. keep both sources visible
3. recommend manual review before changing a synthesis

## Minimal Output

When learning is complete, the response should include:

1. what was learned
2. what was saved
3. what was not generalized
4. local save path
