# Knowledge Base

This directory stores persistent writing knowledge learned from user-provided literature.

## Structure

1. `paper-notes/`: one note per paper
2. `syntheses/`: cross-paper summaries of stable patterns

## Rules

1. Save reusable writing patterns, not raw paper dumps.
2. Keep paper-specific scientific findings scoped to the note that introduced them.
3. Do not overwrite older notes silently.
4. When a new paper conflicts with older guidance, record the conflict and request manual review before changing a synthesis.
5. Load the minimum relevant note or synthesis for a task.

## Paper Note Template

Each paper note should include:

1. citation
2. learning date
3. study context
4. modality stack
5. reusable writing patterns
6. non-generalized content
7. conflict notes
8. suggested use cases

## Synthesis Template

Each synthesis should include:

1. topic
2. source notes used
3. stable patterns seen across papers
4. open conflicts
5. recommended writing applications
